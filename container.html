Pseudo Container

Il giorno 14/07/2024 ho provato a creare una struttura per un container.
Brutalizzando il concetto, e non me ne vogliano troppo i programmatori professionisti; un container è un contenitore di app che si presta
alla loro distribuzione. Ora mi son chiesto perché non sperimentere con le ISO prima di imparare ad usare docker?
Sì vero non sono dei container, dopo varie ricerche da me effettuate, tutta via sono pur sempre degli archivi.


Illustro qui sotto la mia prima idea. Ora non mi ricordo più precisamente quali tutorial ho seguito perciò nessuna fonte.
Presumo abbia attinto pure dalla documentazione di docker oltre che al Microsoft Learns dedicato alla creazione di un SDK per Visual Studio.
E unendo tutti i puntini vorrei provare a costruire una iso contenente:

  • Un container in html e un suo manifesto json
  • UnDTO
  • Un mapper in javascript che comunica col DTO
  • Un'area dedicata allo storage di dati e di applicazioni
  • Delle eventuali query o link per comunicare con eventuali web app
  • Uno script, eventuale e a basso livello, che orchestra il tutto.

L'immagine qui sotto illustra meglio ciò che ho in mente

I Chunk

Il giorno 02/01/2025 ho voluto integrare, ispirato al File System dell'AmigaOS, introdurre il concetto di chunk.

Un chunk è un'immagine ISO di 3gb contendo un form in xml e un auth in xml contenente la struttura dei file archhiviati.
